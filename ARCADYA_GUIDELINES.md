# ARCADYA — הנחיות בנייה אחידות לחבילת משחקי ארקייד

מסמך זה מגדיר את כללי ההקמה, העיצוב, ההתנהגות והמבנה הטכני של חבילת משחקי הארקייד **ARCADYA**.

המטרה היא שכל משחק חדש שייבנה לחבילה יישען על אותם כללים, כדי ליצור חוויית שימוש אחידה, פשוטה, מהירה וברורה.

---

## 1. מהות החבילה

**ARCADYA** היא חבילת משחקי ארקייד קצרים, פשוטים ומהירים, המיועדים לקהל צעיר שלא בהכרח מכיר את משחקי הארקייד הקלאסיים.

המשחקים צריכים להיות:

- פשוטים להבנה תוך כמה שניות.
- רזים טכנית.
- מהירים להפעלה.
- ללא עומס תפריטים.
- ללא הרשמה.
- ללא פרופילים.
- ללא מוזיקת רקע.
- עם שליטה אחידה בכל המשחקים.
- מותאמים קודם כול לטלפון נייד ולטאבלט במצב אנכי.

---

## 2. עקרון Mobile First / Portrait First

החבילה והמשחקים מיועדים בראש ובראשונה למסכים אנכיים:

- טלפון נייד — תצוגת מסך מלא.
- טאבלט — תצוגה אנכית מלאה.
- מחשב — המשחק יוצג במרכז המסך בלבד, בתוך מסגרת אנכית, ולא יימתח לרוחב כל המסך.

במחשב, אזור המשחק צריך לתפוס בערך את השליש האמצעי של המסך, עם שוליים ריקים משני הצדדים.

### הנחיה טכנית

```text
The game must be designed mobile-first and portrait-first.
On desktop screens, the game container must remain centered and limited to a mobile/tablet portrait width, approximately the middle third of the screen.
Do not stretch the game to full desktop width.
```

## 4. אינטרו לוגו בכניסה לחבילה

בכל כניסה של משתמש ל-ARCADYA יופיע אינטרו של הלוגו.

המשתמש יספק קובץ לוגו / אנימציית לוגו / וידאו / GIF / SVG / PNG, ויש לשלב אותו במסך כניסה קצר לפני הצגת מסך המשחקים הראשי.

### כללי האינטרו

- האינטרו מופיע בכל כניסה לחבילת ARCADYA.
- האינטרו מופיע לפני מסך בחירת המשחקים.
- האינטרו אינו מופיע בכל מעבר בין משחקים.
- כאשר חוזרים ממשחק למסך הראשי, לא מציגים שוב את אינטרו הלוגו באותה סשן.
- האינטרו צריך להיות קצר, נקי ולא מעכב.
- משך מומלץ: 2.5 עד 4 שניות.
- לאחר סיום האינטרו יש מעבר אוטומטי למסך הראשי.
- אם קובץ האינטרו הוא וידאו/אנימציה, יש להפעיל אותו ללא דרישה ללחיצה נוספת.
- אם הדפדפן חוסם autoplay עם סאונד, האינטרו צריך לעבוד גם ללא סאונד.

### כפתור דילוג

מומלץ להוסיף כפתור קטן ועדין:

```text
דלג
```

הכפתור יופיע בפינה התחתונה או העליונה, ולא יבלוט מדי.

### התנהגות סשן

יש להשתמש ב-`sessionStorage` כדי לוודא שהאינטרו מוצג פעם אחת בלבד בכל פתיחה/סשן של החבילה.

דוגמה רעיונית:

```js
const INTRO_KEY = 'arcadyaIntroShown';

function shouldShowIntro() {
  return !sessionStorage.getItem(INTRO_KEY);
}

function markIntroShown() {
  sessionStorage.setItem(INTRO_KEY, 'true');
}
```

בטעינת `index.html`:

```js
if (shouldShowIntro()) {
  showLogoIntro();
} else {
  showMainMenu();
}
```

בסיום האינטרו:

```js
markIntroShown();
showMainMenu();
```

### חשוב

האינטרו הוא חלק ממיתוג המוצר, לא מסך הדרכה.  
לא להוסיף בו הוראות משחק, טקסטים ארוכים או כפתורים מיותרים.

## 5. מסך ראשי של ARCADYA

מסך הפתיחה הראשי יופיע לאחר אינטרו הלוגו.

המסך הראשי יכלול:

- שם החבילה: **ARCADYA**.
- כפתורים / כרטיסים לכל המשחקים.
- לכל משחק לוגו או אייקון ברור.
- לחיצה על כרטיס פותחת את המשחק.
- בפינה השמאלית העליונה: כפתור יציאה מהחבילה.

המסך צריך להרגיש כמו מכונת משחקים צעירה, צבעונית ומזמינה.

## 6. יציאה ממסך החבילה

במסך הראשי של ARCADYA יהיה כפתור יציאה בפינה השמאלית העליונה.

בלחיצה עליו תופיע שאלה:

```text
לצאת מ־ARCADYA?
```

אפשרויות:

- לא — חזרה למסך הראשי.

## 7. מבנה מסך קבוע בכל משחק

כל משחק מחולק לשני אזורים:

### אזור המשחק

החלק העליון והמרכזי של המסך.  
שם מתרחש המשחק עצמו: דמויות, כדורים, אויבים, מכשולים, מסלול, ניקוד וכו’.

### אזור כפתורי השליטה

בחלק התחתון של המסך יהיה פס שליטה קבוע.

בין אזור המשחק לבין אזור השליטה יהיה קו גבול ברור והפרדה ויזואלית.

### כללים חשובים

- אזור המשחק לא יגלוש לתוך אזור כפתורי השליטה.
- אובייקטים של המשחק לא יופיעו מתחת לקו הגבול.
- הפס התחתון שמור אך ורק לכפתורי השליטה.
- יש לשמור על מרווחים נוחים לאצבעות.

---

## 8. כפתורי שליטה קבועים

בכל המשחקים יהיו אותם כפתורי שליטה, באותו מיקום.

### בצד ימין של הפס התחתון

ארבעה חצים:

- למעלה
- למטה
- ימינה
- שמאלה

החצים נשארים קבועים בכל המשחקים, גם אם משחק מסוים לא משתמש בכל ארבעת הכיוונים.

### בצד שמאל של הפס התחתון

כפתור עגול אחד — כפתור פעולה.

התפקיד שלו משתנה ממשחק למשחק, למשל:

- קפיצה
- ירייה
- האצה
- בלימה
- איסוף
- החלפת מצב
- פעולה מיוחדת

אבל מבחינת המשתמש, הכפתור העגול תמיד מייצג את הפעולה המרכזית במשחק.

## 9. כפתורי מערכת קבועים בתוך משחק

### יציאה

בפינה השמאלית העליונה של כל משחק יהיה כפתור יציאה.

בלחיצה עליו תופיע שאלה:

```text
לצאת מהמשחק?
```

אפשרויות:

```text
כן
לא
```

- כן — חזרה למסך הראשי של ARCADYA.
- לא — חזרה למשחק.

### פאוז

בפינה הימנית העליונה יהיה כפתור פאוז.

- במצב רגיל יוצג סימן Pause.
- בלחיצה המשחק נעצר.
- הסימן משתנה ל-Play.
- לחיצה נוספת ממשיכה את המשחק בדיוק מאותה נקודה.

כאשר המשחק מושהה, יופיע חיווי במרכז המסך:

```text
המשחק מושהה
```

### סאונד

ליד כפתור הפאוז, בחלק העליון הימני, יהיה כפתור סאונד.

- לחיצה אחת משתיקה את כל הצלילים.
- לחיצה נוספת מחזירה את הצלילים.
- מצב הסאונד נשמר במהלך המשחק.

## 10. חיווי לחיצה אחיד

כל כפתור בכל המשחקים חייב להגיב מיד ללחיצה.

התגובה תכלול:

- שינוי צבע קצר.
- הקטנה קלה של הכפתור או אפקט לחיצה.
- צליל לחיצה קצר, אם הסאונד פעיל.
- חיווי ויזואלי ברור גם כשהסאונד כבוי.

המשתמש תמיד צריך להבין שהלחיצה נקלטה.

## 11. פתיחת כל משחק

כל משחק מתחיל באנימציית פתיחה קבועה של 4 שניות.

במסך הפתיחה יופיע שם המשחק:

- במרכז המסך.
- באותיות עבות.
- מילוי צבעוני.
- קו מתאר או זוהר.
- הבהוב קל.
- סגנון עתידני-ארקיידי, בהשראת שנות השמונים, אבל נקי ומודרני.

לאחר 4 שניות המשחק מתחיל אוטומטית.

אין צורך בכפתור "התחל", אלא אם משחק מסוים ממש מחייב זאת.

## 12. צלילים אחידים

בכל המשחקים יהיה סט צלילים קבוע.

הצלילים:

1. לחיצה על כפתור.
2. הצלחה / איסוף.
3. טעות.
4. פסילה / פגיעה.
5. שיא חדש.
6. פתיחת משחק / מעבר.

מותר להשתמש בחיווי קולי קצר על פעולות, אבל בעדינות, בלי להפוך את המשחק לרועש.

## 13. ניקוד אחיד

בכל משחק יהיה מנגנון ניקוד פשוט.

בחלק העליון של המסך, באזור המשחק, יוצגו לפי הצורך:

- ניקוד נוכחי.
- שיא אישי.
- חיים / זמן — רק אם רלוונטי למשחק.

לא כל משחק חייב להשתמש בחיים או בטיימר, אבל אם יש כאלה, הם יופיעו באותו סגנון ובאותו אזור כללי.

## 15. קושי והתקדמות

הקושי במשחקים צריך לעלות בהדרגה, אבל בצורה פשוטה.

אפשרויות מומלצות:

- כל 20–30 שניות המשחק נהיה מעט מהיר יותר.
- כל כמה הצלחות נוסף מכשול.
- כל שלב מעלה מעט את הקצב.
- כל ניקוד מסוים משנה מעט את האתגר.

לא לבנות מערכת רמות מורכבת.

המטרה היא משחקים רזים, קצרים ומהירים.

## 16. עקרונות טכניים

כל משחק יהיה:

- קובץ HTML עצמאי.
- CSS ו-JS בתוך אותו קובץ, אלא אם נחליט אחרת.
- ללא תלות באינטרנט.
- ללא ספריות חיצוניות.
- מותאם לטאץ'.
- ניתן להרצה בטלפון, בטאבלט ובמחשב.
- Mobile-first.
- Portrait-first.

### הנחיה טכנית כללית

```text
Build each game as a single self-contained HTML file.
All CSS and JavaScript should be inline.
No external libraries.
No internet dependency.
The layout must be portrait-first and mobile-first.
On desktop, center the game in a fixed portrait container instead of stretching it full width.
```

## 17. שמירת מידע מקומית

אין צורך בפרופילים או בהרשמה.

אפשר לשמור מקומית בלבד:

- שיא אישי לכל משחק.
- מצב סאונד.
- אולי תאריך משחק אחרון, אם יידרש בעתיד.

שמירה תתבצע באמצעות `localStorage`.

דוגמה:

```js
localStorage.setItem('arcadya_sound_enabled', 'true');
localStorage.setItem('arcadya_snake_high_score', '1200');
```

## 19. סיכום עקרונות

ARCADYA צריכה להרגיש כמו חבילת מיני-משחקי ארקייד מודרנית, צעירה ומהירה.

המשתמש צריך להבין מיד:

- איפה בוחרים משחק.
- איך זזים.
- מה כפתור הפעולה.
- איך עוצרים.
- איך משתיקים סאונד.
- איך יוצאים.
- איך משחקים שוב.

החוויה צריכה להיות פשוטה, אחידה ורזה, עם מיתוג ברור דרך אינטרו הלוגו, עיצוב ארקיידי, צלילים אחידים ומבנה שליטה קבוע.
